# 部门排班系统 - 功能更新日志

## 版本 2.2 - 接续排班功能 (2026-03-02)

### 🎉 新增功能

#### 1. 接续上一年排班顺序
- ✅ 可选择接续上一年的排班，保持轮换连贯性
- ✅ 自动计算起始组，确保公平轮换
- ✅ 支持手动输入上一年最后值班组号

#### 2. 显示最后值班信息
- ✅ 排班完成后显示最后值班的组和日期
- ✅ 方便用户记录，用于下次接续

### 📝 修改内容

**修改的函数：**

1. `get_user_input()`
   - 新增询问是否接续上一年
   - 新增输入上一年最后值班组号
   - 返回起始组索引

2. `create_schedule()`
   - 新增参数 `start_group_index`（默认值为0）
   - 支持从指定组开始排班

3. `main()`
   - 接收并传递起始组索引
   - 显示最后值班组信息

### 💡 使用示例

```
--- 排班顺序设置 ---
是否接续上一年的排班顺序？(y/n，默认n)：y
请输入上一年最后值班的组号（1-3）：2
提示：本次排班将从第3组开始
```

### 🎯 功能优势

- **公平性**：避免某组总在固定月份值班
- **连续性**：保持多年排班的轮换顺序
- **灵活性**：可选择接续或重新开始
- **易用性**：简单输入即可实现接续

## 版本 2.1 - 调休日处理 (2026-03-02)

### 🎉 新增功能

#### 自动识别和排除调休日
- ✅ 使用 `chinese_calendar.is_workday()` 识别调休日
- ✅ 周末如果是调休上班日，自动排除不排班
- ✅ 符合实际工作情况，减轻员工负担

### 📝 修改内容

**修改的函数：**

1. `get_all_schedule_dates()`
   - 新增调休日检测逻辑
   - 自动跳过调休日

### 💡 使用示例

```
2026年5月排班：
  5月1-5日：劳动节，安排值班 ✅
  5月9日（周六）：调休上班，不排班 ❌
```

## 版本 2.0 - 连休固定组 (2026-03-02)

### 🎉 新增功能

#### 1. 连休假期固定组值班
- ✅ 清明节、端午节、中秋节等连休由同一组值班
- ✅ 劳动节、国庆节特殊分段处理

#### 2. 假期时段识别
- ✅ 自动识别连续假期时段
- ✅ 为时段生成唯一标识符
- ✅ 整个时段分配同一个组

### 📝 新增函数

1. `get_holiday_periods()` - 识别连续假期时段
2. `get_period_identifier()` - 生成时段标识符
3. `find_available_group_for_period()` - 为时段分配组

### 📝 修改内容

**修改的函数：**

1. `create_schedule()`
   - 改为先识别时段，再逐时段分配
   - 确保连休期间使用同一组

### 💡 排班规则

| 假期类型 | 排班方式 |
|---------|---------|
| 清明节连休 | 固定一个组 |
| 端午节连休 | 固定一个组 |
| 中秋节连休 | 固定一个组 |
| 劳动节 | 5月1-2日一组，5月3-5日另一组 |
| 国庆节 | 10月1-3日一组，10月4-7日另一组 |
| 普通周末 | 正常轮流 |

## 版本 1.0 - 基础功能 (初始版本)

### 🎉 核心功能

1. ✅ 自动识别周末和法定节假日
2. ✅ 每组每月值班不超过4天
3. ✅ 避免连续值班
4. ✅ 组员固定，整组值班
5. ✅ 导出美观的Excel排班表

### 📝 主要函数

1. `get_user_input()` - 获取用户输入
2. `get_all_schedule_dates()` - 获取排班日期
3. `create_schedule()` - 创建排班表
4. `find_available_group()` - 找到可用组
5. `export_to_excel()` - 导出Excel
6. `main()` - 主函数
