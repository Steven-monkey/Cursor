# 接续排班功能说明

## 📋 功能概述

接续排班功能允许新一年的排班从上一年结束的组继续轮换，确保各组值班机会的公平性和连续性。

## 🎯 为什么需要接续功能？

### 问题场景

**没有接续功能时：**

```
2025年排班：
  1月：第1组 → 第2组 → 第3组 → ...
  12月：... → 第2组（最后）

2026年排班（不接续）：
  1月：第1组 → 第2组 → 第3组 → ...
  
问题：
  - 第1组连续两年都在1月值班
  - 第2组在2025年12月和2026年1月之间间隔太短
  - 轮换不公平
```

**使用接续功能后：**

```
2025年排班：
  1月：第1组 → 第2组 → 第3组 → ...
  12月：... → 第2组（最后）

2026年排班（接续）：
  1月：第3组 → 第1组 → 第2组 → ...
  
优点：
  - 各组值班时间分布均衡
  - 轮换顺序连贯
  - 更加公平合理
```

## 🚀 使用方法

### 步骤1：完成上一年排班

运行程序生成上一年的排班表：

```bash
python 部门排班系统.py
```

排班完成后，程序会显示：

```
排班统计信息
==================================================
第1组：共值班 42 天
第2组：共值班 41 天
第3组：共值班 43 天

最后值班：第2组（2025年12月28日）
提示：下次排班时可选择接续此次排班顺序

排班完成！
```

**重要：记录"最后值班"的组号（本例中是第2组）**

### 步骤2：新一年排班时接续

运行程序开始新一年的排班：

```bash
python 部门排班系统.py
```

按提示输入信息：

```
--- 排班顺序设置 ---
是否接续上一年的排班顺序？(y/n，默认n)：y
请输入上一年最后值班的组号（1-3）：2
提示：本次排班将从第3组开始
```

系统会自动从第3组开始排班（2的下一组）。

## 🔧 技术原理

### 计算逻辑

```python
# 假设有3个组，上一年最后是第2组
last_group = 2        # 上一年最后值班的组号
group_count = 3       # 总组数

# 计算起始组索引
start_group_index = last_group % group_count
# 结果：2 % 3 = 2（即第3组，索引为2）

# 如果上一年最后是第3组
last_group = 3
start_group_index = last_group % group_count
# 结果：3 % 3 = 0（即第1组，索引为0）
```

### 代码实现

**修改的函数：**

1. **`get_user_input()`**
   - 新增询问是否接续上一年
   - 如果接续，询问上一年最后值班组号
   - 计算起始组索引
   - 返回起始组索引

2. **`create_schedule()`**
   - 新增参数 `start_group_index`
   - 从指定的组开始轮换排班

3. **`main()`**
   - 接收起始组索引
   - 传递给排班函数
   - 显示最后值班的组（用于下次接续）

## 📊 效果对比

### 示例：3个组，每年约120天需要排班

#### 不接续的情况

| 年份 | 1月开始组 | 12月结束组 | 第1组总天数 | 第2组总天数 | 第3组总天数 |
|------|----------|-----------|------------|------------|------------|
| 2024 | 第1组 | 第2组 | 41 | 40 | 39 |
| 2025 | 第1组 | 第2组 | 41 | 40 | 39 |
| 2026 | 第1组 | 第2组 | 41 | 40 | 39 |

**问题：**
- 第1组每年都在1月值班（春节期间）
- 第3组每年都在12月值班（年底）
- 不够公平

#### 接续的情况

| 年份 | 1月开始组 | 12月结束组 | 第1组总天数 | 第2组总天数 | 第3组总天数 |
|------|----------|-----------|------------|------------|------------|
| 2024 | 第1组 | 第2组 | 41 | 40 | 39 |
| 2025 | 第3组 | 第1组 | 40 | 41 | 39 |
| 2026 | 第2组 | 第3组 | 39 | 40 | 41 |

**优点：**
- 各组在不同月份值班，分布均衡
- 3年为一个完整轮换周期
- 更加公平合理

## 💡 使用建议

### 何时使用接续功能

✅ **建议使用：**
- 每年年初排班时
- 组员结构稳定
- 追求公平轮换

❌ **不建议使用：**
- 第一次使用系统
- 组数发生变化
- 组员大幅调整
- 想要重新开始轮换

## 📝 常见问题

### Q1: 忘记上一年最后是哪个组怎么办？

**A**: 
1. 打开上一年的排班表Excel文件
2. 查看最后一行的"值班组"列
3. 如果找不到文件，选择'n'从第1组开始

### Q2: 输入错误的组号会怎样？

**A**: 
- 程序会验证组号是否在有效范围内
- 如果无效，会显示警告并从第1组开始
- 不会导致程序崩溃

### Q3: 组数变化了还能接续吗？

**A**: 
- 不建议接续
- 组数变化会影响轮换逻辑
- 建议选择'n'重新开始

### Q4: 接续后会影响其他排班规则吗？

**A**: 
- 不会影响
- 只改变起始组
- 其他规则（月度限制、避免连续等）正常生效

### Q5: 可以跳过某些组吗？

**A**: 
- 当前版本不支持
- 所有组都会参与轮换
- 如需特殊安排，可以手动修改生成的Excel文件

### Q6: 接续功能适用于跨年排班吗？

**A**: 
- 适用
- 如果排班跨年（如2025年10月-2026年3月）
- 下次排班时仍可接续
