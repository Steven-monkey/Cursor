# 调休功能说明

## 📅 什么是调休？

在中国，为了让法定节假日连休，政府会将节假日前后的周末调整为工作日。这些需要上班的周末就是**调休日**。

### 举例说明

**2026年劳动节（5月1-5日放假）：**
- 5月1日（周五）：法定节假日 ✅ 休息
- 5月2日（周六）：周末+节假日 ✅ 休息
- 5月3日（周日）：周末+节假日 ✅ 休息
- 5月4日（周一）：法定节假日 ✅ 休息
- 5月5日（周二）：法定节假日 ✅ 休息
- **5月9日（周六）：调休日** ❌ **需要上班（不排班）**

**2026年国庆节（10月1-7日放假）：**
- 10月1-7日：节假日 ✅ 休息
- **10月10日（周六）：调休日** ❌ **需要上班（不排班）**

## 🎯 排班系统如何处理调休？

### 自动识别调休日

程序使用 `chinese_calendar` 库自动识别调休日：

```python
# 判断是否为工作日（包括调休日）
is_workday = chinese_calendar.is_workday(current_date)

# 判断是否为调休日：周末但需要上班的日期
is_makeup_workday = is_weekend and is_workday
```

### 排除调休日

如果某个日期是调休日，程序会：
1. ✅ 识别该日期为周末
2. ✅ 识别该日期需要上班（调休）
3. ✅ **自动跳过，不安排值班**

## 💡 为什么要排除调休日？

### 原因分析

1. **调休日需要上班**
   - 员工在调休日需要正常工作
   - 不应该再安排额外的值班任务

2. **公平性考虑**
   - 调休日已经需要上班，再值班会增加负担
   - 应该让员工在真正的休息日值班

3. **符合实际情况**
   - 调休日办公室正常运转，有人在岗
   - 不需要额外安排值班人员

## 🔧 技术实现

### 核心代码

```python
# 使用chinese_calendar库判断是否为工作日（包括调休日）
is_workday = chinese_calendar.is_workday(current_date)

# 判断是否为调休日：周末但需要上班的日期
is_makeup_workday = is_weekend and is_workday

# 如果是调休日，跳过不排班
if is_makeup_workday:
    current_date += timedelta(days=1)
    continue
```

### chinese_calendar 库的功能

- `is_holiday(date)`: 判断是否为法定节假日
- `is_workday(date)`: 判断是否为工作日（包括调休日）
- 自动识别中国的法定节假日和调休安排
- 数据每年更新，支持最新的放假安排

## 📝 常见问题

### Q1: 如何知道哪些日期是调休日？
**A**: 程序自动识别，无需手动输入。`chinese_calendar` 库包含最新的调休安排。

### Q2: 如果想在调休日也排班怎么办？
**A**: 修改代码，删除或注释掉调休日检查的部分（第89-92行）。

### Q3: 调休安排每年都不一样，程序能自动更新吗？
**A**: 需要更新 `chinese_calendar` 库到最新版本：
```bash
pip install --upgrade chinese_calendar
```

### Q4: 如何查看某个日期是否为调休日？
**A**: 可运行主程序指定月份，查看生成的排班表中是否包含该日期。
